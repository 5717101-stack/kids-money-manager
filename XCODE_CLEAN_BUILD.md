# עדכון גרסה באייפון - הוראות

## הבעיה
אחרי עדכון גרסה, האפליקציה עדיין מציגה את הגרסה הישנה.

## הפתרון

### שלב 1: Clean Build ב-Xcode
1. פתח את הפרויקט ב-Xcode:
   ```bash
   open ios/App/App.xcworkspace
   ```
   או
   ```bash
   open ios/App/App.xcodeproj
   ```

2. ב-Xcode:
   - לחץ על **Product** → **Clean Build Folder** (או `Cmd+Shift+K`)
   - זה ינקה את כל ה-cache של ה-build

### שלב 2: Build מחדש
1. ב-Xcode:
   - לחץ על **Product** → **Build** (או `Cmd+B`)
   - זה יבנה את האפליקציה מחדש עם הגרסה החדשה

### שלב 3: הסר והתקן מחדש
1. **הסר את האפליקציה מהמכשיר:**
   - לחץ לחיצה ארוכה על האייקון של האפליקציה
   - בחר "הסר אפליקציה"
   - אישר את ההסרה

2. **התקן מחדש:**
   - ב-Xcode, לחץ על **Product** → **Run** (או `Cmd+R`)
   - או גרור את האפליקציה מהמכשיר והתקן מחדש

### שלב 4: בדוק את הגרסה
1. פתח את האפליקציה
2. לך לדף כלשהו (כמו Welcome Screen או OTP)
3. בדוק את הגרסה בתחתית הדף
4. אמור להיות: **גרסה 3.4.77**

## אם עדיין לא עובד

### אפשרות 1: נקה את Derived Data
1. ב-Xcode:
   - **Xcode** → **Preferences** → **Locations**
   - לחץ על החץ ליד "Derived Data"
   - מחק את התיקייה של הפרויקט
   - בנה מחדש

### אפשרות 2: נקה את Build Cache
```bash
# נקה את ה-cache של Capacitor
rm -rf ios/App/App/public
npm run build
npx cap sync ios
```

### אפשרות 3: בדוק את הגרסה ב-Xcode
1. ב-Xcode, בחר את הפרויקט
2. בחר את ה-Target (App)
3. לך ל-**General** → **Identity**
4. בדוק את **Version** - אמור להיות **3.4.77**

## הערות
- לפעמים צריך להסיר את האפליקציה מהמכשיר כדי לראות את הגרסה החדשה
- ה-cache של iOS יכול לשמור את הגרסה הישנה
- Clean Build הוא חשוב מאוד אחרי עדכון גרסה
